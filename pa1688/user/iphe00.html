IPH E00 IP Phone
 
Vida IPH E00 IP phones do not use our standard reference design, all need their special software to work. However, as their developers shared all hardware and a large portion of software changes with us, sometimes we might be able to provide update software for those phones, the following is a successful example.
There are at least 2 phone case and 5 different designs in the IPH E00 family, the one showed here is IPH_E00_2G and use software name such as iph_e00_2g_sip_us_165002.bin for update.
Web manual: SIP IAX2 H323 MGCP. Menu manual: SIP IAX2 H323 MGCP. Telnet manual: SIP IAX2 H323 MGCP.
Front. Do not make judgement of an IP phone by its phone case. Most manufacturers use ready PSTN phone case for their IP phone products, and the case showing here are the most heavily used. We know at least 6 different PA1688 manufacturers used it, for example Primeworx P100. Maybe because it is the almost the cheapest one. Large
Back. Large
PCB. The text on the MX chip reads 29LV008BTC-70. When using PalmTool, remember to select PFlash type to AM29LV008B. This is the most complicated PA1688 hardware board ever built. Besides ethernet, it also has modem chip connected to PA1688 8051 data bus via 16c550. ATCOM and Highlink also made dialup IP phone based on PA1688 later, with the help of Davicom engineers, they connected 8051 data bus directly to the 16c550 interface inside the modem chip and saved the outside 16c550 chip.
